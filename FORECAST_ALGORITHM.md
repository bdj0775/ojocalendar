# 예약 점유율 예측 알고리즘 설명서

> 대상: 숙소 운영자 (비개발자)
> 한 줄 요약: **"지금까지 들어온 예약 + 앞으로 들어올 예약을 과거 패턴과 비교해서 월말 점유율을 추정합니다."**

---

## 예측이 하는 일 — 쉬운 비유

날씨 예보와 똑같습니다.

기상청은 오늘의 온도·기압·습도를 측정한 뒤, 과거 데이터와 비교해서 "내일 비 올 확률 70%"를 계산합니다. 비가 올 확률은 오늘 측정하든 3일 전에 측정하든 그 시점에 맞는 계산을 합니다.

이 알고리즘도 마찬가지입니다.
- **언제 보든 그 시점의 예약 현황을 측정하고**
- **그 시점 기준의 과거 패턴과 비교해서**
- **월말 최종 점유율을 추정합니다**

2달 전에 확인하든, 1주일 전에 확인하든, 당일에 확인하든 — 항상 그 시점에 맞는 예측값이 나옵니다.

---

## 예측에 사용되는 재료 4가지

### 재료 1 — 현재 확정 예약 (OTB)

오늘 기준으로 이미 예약 확정된 박 수입니다.

> 예: 6월 30일 중 24일이 확정 → OTB = 80%

**예측값은 절대로 이 수치 아래로 내려가지 않습니다.** 확정된 예약이 사라질 수 없으니까요.

---

### 재료 2 — 예약 도착 속도 (τ, 타우)

예약이 얼마나 빨리 들어오는지를 수치로 표현한 값입니다. 입실일에 가까워질수록 예약이 급격히 늘어나는 패턴이 있는데, 그 기울기를 측정합니다.

```
입실 90일 전 ──────────────░░░░░░░░░░  (아직 예약이 적음)
입실 30일 전 ──────────────────────░░  (절반 이상 찬 상태)
입실  7일 전 ────────────────────────  (거의 다 찬 상태)
```

- τ가 작을수록 → 손님들이 일찍 예약 (얼리버드 고객 많음)
- τ가 클수록 → 손님들이 임박해서 예약 (충동 예약 고객 많음)

**이 값은 최근 12개월 실제 예약 데이터로 자동 계산됩니다.** 데이터가 쌓일수록 더 정확해집니다.

---

### 재료 3 — 과거 같은 달 실적

해당 월과 동일한 달(전년도, 재작년)의 최종 점유율을 참고합니다.

| 비교 대상 | 내용 | 반영 비중 |
|---|---|---|
| 전년 동월 (STLY) | 작년 같은 달 최종 점유율 | 최대 40% |
| 2년 전 동월 | 재작년 같은 달 최종 점유율 | 최대 30% |
| 현재 예약 속도 | 지금 예약 페이스 기반 추정 | 30% |

> ⚠️ **오픈 초기(2025년 6~9월) 데이터는 자동 제외됩니다.**
> 막 시작한 숙소는 아직 시장 안착 전이라 그 시기 데이터를 기준으로 삼으면 예측이 왜곡됩니다.
> 2026년 10월부터는 전년(2025년 10월~) 데이터가 신뢰 가능한 수준으로 편입됩니다.

전년도/재작년 데이터가 없거나 신뢰하기 어려운 경우에는 최근 6개월 평균으로 자동 대체됩니다.

---

### 재료 4 — 전체 구간 백테스트 편향 교정

알고리즘이 과거에 얼마나 틀렸는지 스스로 측정하고, 그 오차를 현재 예측에 보정해주는 기능입니다.

**핵심 아이디어: 과거 완료 월을 "타임머신처럼" 재현합니다**

예를 들어 작년 12월이 완료됐다면:
- "12월 시작 90일 전 시점엔 OTB가 얼마였을까?" → bookingDate로 역산
- "12월 시작 89일 전 시점엔?" → 역산
- "12월 시작 1일 전 시점엔?" → 역산
- 이렇게 **하루 단위로** 90일치 시뮬레이션 → 각 시점마다 "예측 vs 실제" 오차 계산

이것을 모든 완료 월에 대해 반복해서 **오차 커브**를 만듭니다.

```
입실 90일 전 기준 평균 오차: +15%p  (멀리서 볼수록 더 많이 틀림)
입실 60일 전 기준 평균 오차: +11%p
입실 30일 전 기준 평균 오차:  +8%p
입실 14일 전 기준 평균 오차:  +6%p
입실  1일 전 기준 평균 오차:  +4%p
```

지금 예측을 확인하는 시점이 언제든 — 오늘이 입실 2달 전이든, 3일 전이든 — **그 시점에 맞는 오차값을 찾아서 자동으로 보정**합니다.

---

### 보너스 — 상대 페이스 비교

같은 "입실까지 N일 전" 시점을 기준으로, 이번 달 OTB가 과거 평균보다 얼마나 높은지 비교합니다.

```
과거 "입실 20일 전" 시점 평균 OTB = 70%
현재 "입실 20일 전" 시점 OTB      = 84%
─────────────────────────────────────
상대 페이스 = 84 ÷ 70 = 1.2배
→ 앞으로 들어올 예약도 1.2배 많을 것으로 추정
```

이를 통해 "5월보다 6월 예약 속도가 빠른데 5월은 100% 마감했다" 같은 신호가 반영됩니다.

## 예측이 시간이 지날수록 정확해지는 이유

| 운영 기간 | 상태 |
|---|---|
| 오픈 직후 0~4개월 | 예측값 표시 안 함. 데이터 신뢰 불가. |
| 6~12개월 | 기초 예측 시작. 편향 커브 구축 중. |
| 12~18개월 | 편향 커브 본격 작동. 정확도 향상. |
| 24개월 이상 | 계절성 패턴까지 학습. 고정밀 예측. |

세 가지 수치가 자동으로 개선됩니다:

1. **τ (예약 도착 속도)** → 매달 재계산
2. **편향 커브 (D=0~90 전체)** → 완료 월이 쌓일수록 각 구간의 샘플이 늘어나 정밀해짐
3. **전년 동월 신뢰도** → 2026년 10월부터 신뢰할 수 있는 STLY 데이터 등장

---

## 편향 커브가 실제로 어떻게 쌓이는가

완료 월 하나가 추가될 때마다 커브에 **91개의 데이터 포인트**가 추가됩니다.

```
완료 월 7개 기준:  7 × 91일 = 637개 샘플 → 커브 완성
완료 월 12개 기준: 12 × 91일 = 1,092개 샘플 → 더 정밀
완료 월 24개 기준: 24 × 91일 = 2,184개 샘플 → 계절성까지 반영
```

이 커브는 앱 실행 시 **이미 과거 실데이터로 즉시 계산됩니다.** 앞으로의 데이터를 기다릴 필요 없이 지금 DB에 있는 예약들의 bookingDate를 역산해서 과거 각 시점의 OTB를 재구성합니다.

## 예측값 해석 가이드

| 표시 | 의미 |
|---|---|
| **OTB = 예측** (예: OTB 80%, 예측 80%) | 더 채워질 근거가 없음. 현재가 최선 추정치. |
| **예측 > OTB 조금** (예: 80% → 88%) | 과거 패턴과 속도상 추가 예약이 들어올 가능성 높음. |
| **예측 > OTB 많이** (예: 60% → 85%) | 예약이 임박해서 몰리는 달. 지금은 비어 보여도 결국 찰 가능성. |
| **예측 = 100%** | 과거 패턴 + 편향 보정 결과 만실이 거의 확실한 상태. |

---

## 예측이 빗나갈 수 있는 상황

- **갑작스러운 외부 이벤트** — 근처 행사, 자연재해, 명절 연휴 변경
- **가격 정책 변경** — 갑자기 요금을 올리거나 내릴 때
- **대규모 취소** — 예약 취소가 몰릴 경우 (이미 확정된 예약 기준이라 반영 안 됨)
- **오픈 초기 한계** — 아직 12개월치 데이터가 전부 신뢰 가능하지 않음

## 자주 묻는 질문

**Q: "월말 예상 90%"는 얼마나 믿어야 하나요?**
> 현재 OTB(확정 예약)는 사실입니다. 그 위로 올라가는 예측 부분은 과거 패턴 기반 추정이므로 참고용입니다. 신뢰도 %가 낮으면 불확실성이 크다는 뜻이고, 높으면 과거 패턴이 일관성 있게 뒷받침한다는 뜻입니다.

**Q: 예측이 계속 OTB랑 똑같이 나와요.**
> 과거 데이터가 부족하거나, 현재 OTB가 이미 과거 최고치를 넘어서 더 올릴 근거가 없는 경우입니다. 데이터가 쌓이면 자연스럽게 개선됩니다.

**Q: 예측값이 갑자기 바뀌었어요.**
> 새 달이 완료되면서 편향 커브가 업데이트됐거나, τ(예약 속도)가 재계산된 것입니다. 정상 동작입니다.

**Q: 6월 예약 속도가 5월보다 빠른데 왜 예측이 낮게 나왔었나요?**
> 기존에는 과거 특정 시점 1개의 스냅샷만 참고했습니다. 이제 D=0~90 전체 구간의 편향 커브를 사용하므로 어떤 시점에 확인하더라도 더 많은 과거 데이터를 반영한 예측이 나옵니다.

**Q: 앱을 켤 때마다 예측이 달라질 수 있나요?**
> 네. 편향 커브는 앱 실행(또는 예약 데이터 변경) 시마다 재계산됩니다. 새 예약이 들어오거나 완료 월이 추가되면 커브가 업데이트되어 예측도 미세하게 바뀔 수 있습니다.
